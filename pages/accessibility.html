Accessibility

Explain how you considered accessibility in your work.

  • Contrast: Changing the colors of the webpage to ensure greater contrast
  • Night Mode: Adding a dark mode theme toggle to reduce visual strain for users who prefer darker interfaces or use the site during evening hours
  • Semantic HTML: Using proper HTML semantic elements (header, nav, main, section, footer) for better screen reader navigation
  • Alt Text: Providing descriptive alt text for all images to ensure screen reader users understand visual content
  • Keyboard Navigation: Ensuring all interactive elements are accessible via keyboard, with visible focus indicators
  • Link Text: Using descriptive link text that makes sense out of context (avoiding "click here" links)
  • Readable Typography: Using web-safe sans-serif fonts with sufficient line spacing (1.6) for improved readability
  • Responsive Design: Building responsive layouts that work across different screen sizes and orientations
  • Form Accessibility: Properly labeling all form inputs and providing clear error messages
  • Image Encoding: Using proper URL encoding for image filenames to ensure they load correctly across different browsers and systems
  • Clear Navigation: Providing consistent navigation patterns and clear page structure throughout the site
